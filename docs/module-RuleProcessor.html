<static> fireRuleEvent(ast, data, processingComplete)

Creates a rule event from the given ast and data. It's basically a special purpose mixin.
Parameters:
Name Type Description
ast module:AST.node The ast associated with the event that will be queried for the base event information
data Object The data to mixin the base event information in to. This object is modified
processingComplete Boolean Indicates if this rule has been processed or not. Useful for doing pre vs post order traversals. Note: every rule fires a rule event twice, once before processing has begun and once after processing has completed, as indicated by this property.
